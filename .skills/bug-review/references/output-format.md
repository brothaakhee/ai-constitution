# Bug Review Output Format

Return confirmed findings first.

## Confirmed Finding Template
1. Severity: short title
Bug type: product, logic, security, data, reliability, or operational
Location: file, system, flow, or artifact
Impact: what breaks for the user or system
Evidence: logs, repro, code path, or artifact
Why this is a bug: referee-approved reasoning
Challenge considered: strongest challenge and why it was rejected
Confidence: high, medium, or low

## Follow-on Sections
- Disputed items
- Rejected items worth monitoring
- Assumptions
- Evidence gaps
- Recommended next verification steps

Keep the report concise and evidence-driven.
